# 🐞 Bug Report

## Summary

The application allows the user to attempt creating a duplicate course category without displaying any validation message or feedback.

> Example: Creating an already existing course category does not show a "Category already exists" validation message.

## Preconditions

- User is logged into the application.
- User has access to **Dynamic Field Management**.
- At least one course category already exists.

---

## Steps to Reproduce

1. Log in to the LMS application with valid credentials.
2. Navigate to **Dynamic Field Management**.
3. Select the **Course Category** option.
4. Click the **Add Category** button.
5. Enter a category name that already exists.
6. Click the **Save/Add** button.

---

## Expected Result

The application should prevent duplicate category creation and display a validation message such as **"Category already exists"** or **"Duplicate categories are not allowed."**

---

## Actual Result

No validation message is displayed, and no appropriate feedback is provided when attempting to create a duplicate course category.
