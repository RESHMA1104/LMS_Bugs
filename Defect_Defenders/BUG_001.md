# 🐞 Bug Report

## Summary

The email displayed in the user's profile does not match the email used during login. After logging in with a valid account, the profile shows a different user's email address.

> Example: Logged in with **testing4@gmail.com**, but the profile displays **testing1@gmail.com**.

## Steps to Reproduce

1. Navigate to the LMS application.
2. Enter the email **testing4@gmail.com**.
3. Enter the password **123**.
4. Click the **Login** button.
5. Click on the **Profile Avatar**.
6. Observe the displayed email address.

## Expected Result

The Profile section should display the same email address that was used to log in (**testing4@gmail.com**).

---

## Actual Result

The Profile section displays **testing1@gmail.com** instead of the logged-in user's email.
