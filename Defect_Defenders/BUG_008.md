# 🐞 Bug Report

## Summary

The **Delete** functionality for Course Categories does not work consistently. After confirming the deletion, the category is not removed, and the application remains in the same state.

> Example: Clicking the **Delete** button on the confirmation dialog does not delete the selected course category.

## Preconditions

- User is logged into the LMS application.
- User has access to **Dynamic Field Management**.
- At least one course category exists.

---

## Steps to Reproduce

1. Log in to the LMS application with valid credentials.
2. Navigate to **Dynamic Field Management**.
3. Select the **Course Category** option.
4. Search for an existing course category.
5. Open the action dropdown for the category.
6. Select **Delete**.
7. Click the **Delete** button in the confirmation dialog.
8. Observe the result.

---

## Expected Result

The selected course category should be deleted successfully and removed from the list.

---

## Actual Result

The Delete confirmation dialog closes or remains in the same state, but the selected course category is not deleted.
