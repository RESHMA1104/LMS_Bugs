# 🐞 Bug Report

## Summary

The application allows users to create duplicate pedagogy elements without displaying any validation message, resulting in duplicate entries.

> Example: Entering an existing pedagogy element name allows it to be added again.

## Preconditions

- User is logged into the application.
- User has access to **Dynamic Field Management**.
- At least one pedagogy element already exists.

---

## Steps to Reproduce

1. Log in to the LMS application with valid credentials.
2. Navigate to **Dynamic Field Management**.
3. Select the **Pedagogy** option.
4. Click the **Add Element** button.
5. Enter a pedagogy element name that already exists.
6. Click the **Save/Add** button.

---

## Expected Result

The application should prevent duplicate pedagogy elements from being created and display an appropriate validation message such as **"Element already exists."**

---

## Actual Result

The application allows duplicate pedagogy elements to be created without displaying any validation message.
